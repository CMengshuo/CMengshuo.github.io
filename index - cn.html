目前项目

目前我正在用Unity开发一款第三人称动作RPG游戏，我会不定期的在这里更新我的项目进度。 你也可以在我的Trello Board上关注该项目详细进展。

足部ik放置系统

实现人物在静止和运动中足部与复杂环境的自然贴合。该系统会自动调节碰撞胶囊，确保人物在静止时能正确的放置双脚并避免骨骼变形。ik权重会根据运动的状态而改变，避免不自然的足部动作。

反应式近战系统 - 格挡，击中，闪避

模仿巫师3做的反应式近战系统。该系统会分析对方的攻击部位参数，在动作混合树执行相应的动作。未来计划是将击中mecanim和ragdoll混合做出更自然的受伤动作以及使用ik做出更加真实的格挡机制。

扩展 - 物品及背包编辑器

我开发的一款扩展编辑器。主要应用于实时模式及编辑模式下对于游戏物品及背包的数据改动。该工具依赖工程对场景物体和prefab的独特id对数据进行序列化。

此扩展支持以下操作：

  • 数据序列化以支持游戏的存档和读档功能
  • 在unity实时及编辑模式下对游戏物品及背包数据进行修改
  • 确保删除物品数据时同时删除所有关联数据，避免数据错误
  • 自动处理物体模型并正确的显示在背包的物品预览界面中
  • 支持对prefab预设背包默认物品
  • 自动创建，修改游戏物品prefab文件，并进行分类存放
  • 自动关联prefab数据：对prefab的重命名及存放位置修改不会影响物品对prefab的依赖
  • 自动添加相应类别的物品类逻辑，比如防具类的骨骼贴合，使物品能被正确使用

战斗系统 - 近距离法术

法术系统目前被集成在背包系统中，因此目前被视为“物品”，能被物品及背包编辑器编辑。法术系统不是我现在的重点开发方向，以后会被分离出来。

交易及战利品系统

交易和战利品系统是背包系统的扩展：系统会读取被交互物体的背包信息并处理。一切有背包组件的物体都可以被玩家交互。目前支持以下功能：

  • 与NPC交易
  • 从NPC身上掠夺，偷窃物品
  • 与环境中的容器进行交互
  • 自动卸下被卖出，偷盗，掠夺和丢弃的装备

背包系统

仿上古卷轴：天际SkyUI mod做的列表式背包系统。具备传统背包系统的基本功能。

  • 装备和卸下装备
  • 捡起和丢弃装备
  • 自动识别玩家在场景中拖拽到人物身上的装备，进行骨骼匹配并添加到人物背包中

近战系统 - 锁定目标

类似于巫师3的目标锁定系统，但玩家可手动切换并解除锁定目标。锁定后人物控制系统以被锁定目标为控制坐标系进行动作混合。AI也采用该锁定系统执行动作逻辑。

角色控制系统

主要动作包来自于 (Longsword Animset Pro)。 该包不带控制器，所以本项目的控制系统及摄像机逻辑是由我开发。主要功能如下：

  • 摄像机的跟随，旋转，放大和缩小
  • 摄像机的室内碰撞检测及处理
  • 适用于人形生物的Mecanim的混合树控制器

一些我以前做过的项目

基于电脑摄像头的机器学习迷你游戏

C++ Ogre3D OpenCV Machine Learning

我基于Ogre3D渲染引擎开发的一款迷你机器学习程序。该程序依赖电脑摄像头在游戏前通过截频记录用户对于不同控制定义的图像，通过提取特征向量并使用SVM分类储存数据。 在游戏中，程序会根据所观察到物体进行数据权重匹配来执行相应的操作。因为基于Ogre3d，该游戏的物理检测和碰撞模拟，粒子系统和光照均由我实现。

InkMan

Unity C# Spine

由我设计和开发的一款2D无限奔跑游戏。该游戏的特点是物理笔触：玩家在屏幕上画出的笔触会与环境产生物理碰撞，从而帮助玩家克服障碍物。 游戏中所有地形，怪物和物品均为实时随机生成，确保无缝的游戏体验。

NanoShock

Unity C#

这款躲避生存类游戏由Brian Moore设计并创作游戏资源，由我开发完成。 在游戏中玩家可以装备不同功能的能量球使自己更为强大，从而生存更久。

Melty Ice

Unity C#

这是我开发的第一款Unity游戏。美术资源和游戏设计来自Brian Moore。 我在这项工程中确定了产品的工作流程和工具使用，使我的组员能使用第三方工具制作关卡地图并导入Unity发布而无需我的代码维护。
