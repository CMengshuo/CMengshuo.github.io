Current Project

I am currently developing a 3rd person ARPG in Unity, you can follow the progress of this project through my Trello.

Foot Placement System

I wrote my own foot placement system. This system should be able to guide the player to walk through some rough terrains. My thinking process here is to adjust the positions of the avatar and its collider to prevent deformation of bones.

Combat System - Block, Dodge, and Hit

My vision for this project is to create an immersive and realistic battle experience. So agents should be able to react to events accordingly. This system would analyze enemies' attacking patterns, then blend animations using calculated parameters. It is my plan to blend mecanim with ragdoll for better hit animation and use IK for more accurate blocking mechanism.

Development Tool (Item, Inventory)

Tools are always important for game development. I developed this editor extension to support data serialization, and easier data manipulation. Not only can you add inventory and item to a scene object, but you can also add them to a prefab: Instantiated objects will have preset items in their inventory. New items can be created through this tool, regardless of the state of the game. In particular, my code will process the model of the item on creation, and show it properly in the inventory camera. What I have in mind is to create a tool to add gameplay features without coding and further editing.

Key features of this tool are as follows:

  • Serialization of items and inventories data to support save and load games
  • Create, add, modify, delete items and inventories databases in editor mode and playmode
  • Mechanisms to safely delete items that are already equipped or in inventory
  • Auto classify and organize item prefab files for the project

Combat System - Spells

Spell system is integrated into my inventory UI. Players can access their skills under category of spells in their inventory. More spells will be added, though magic system is not among my highest priority at the moment.

Loot and Trade System

Loot and trade systems are extensions of inventory system: It simply reads and processes inventory data of other agents. In addition, any object with inventory can be interacted. Key features are as follows:

  • Trade items with NPC
  • Loot items from NPC
  • Loot or store items in containers
  • Auto unequip looted, sold and dropped items

Inventory System

I am a big fan of The Elder Scroll V: Skyrim, so I implemented a similar inventory system, which I like very much. I have two XML files to manage inventory data: One to keep record of items of an inventory and its owner. The other one contains information of each item. The inventory system will parse both files and process the data when needed. This design ensures the implementation of various gameplay features that we see in Skyrim. For personal inventory, these features have been implemented:

  • Equip or unequip items
  • Pick up or drop items
  • Auto recognize attached equipments in scene, add them to inventory and equip them if possible

Combat System - Lock On Target

Free movement gives players more control over an avatar. It also offers players great satisfaction and sense of achievements when they beat the opponents with stunning combo. But it can cause frustration for casual players. The Witcher 3: Wild Hunt uses a target lock on system which requires relatively less control. Though some people find it hard to use, I find it satisfying. My solution for this dilemma is to seek a balance. Players can manually lock on and switch targets, they can also break free from the lock on system anytime they want.

Character Control System

The animation package I was using came without a controller (Longsword Animset Pro). So I designed and implemented my own character control system in Unity animator. The camera logic was added after the control system is functional. My system covers basic features of an ARPG:

  • Camera follow
  • Camera rotate, zoom in and zoom out
  • Indoor collision check for camera
  • Mecanim controller for humanoid character
